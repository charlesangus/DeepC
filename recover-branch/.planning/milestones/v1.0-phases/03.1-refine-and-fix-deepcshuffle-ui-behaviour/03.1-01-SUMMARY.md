# Phase 03.1 Plan 01: Logic Bug Fixes Summary

**ShuffleMatrixWidget radio enforcement scoped per output group, const:0/1 columns added, and identity routing written silently on new-node first open**

## Performance

- **Duration:** ~4 min
- **Started:** 2026-03-16T06:17:46Z
- **Completed:** 2026-03-16T06:21:46Z
- **Tasks:** 2
- **Files modified:** 4

## Accomplishments

- Added `initializeState()` to `ShuffleMatrixKnob` ‚Äî writes `_matrixState` directly without calling `changed()`, preventing recursive widget rebuilds during identity init
- Fixed radio enforcement scope bug: `onCellToggled` now takes `outputGroup` as first parameter and the enforcement loop checks `otherOutputGroup == outputGroup`, so out1 and out2 rows with identically-named channels stay independent
- Added `if (!checked) return;` guard in `onCellToggled` ‚Äî clicking an already-checked button (which fires `toggled(false)`) is now a no-op
- Added `const:0` and `const:1` source columns to the matrix grid, positioned between in1 and in2 with "const" group header spanning both
- Added identity routing initialisation in `syncFromKnob()` ‚Äî on first open when state is empty, each out1 channel is mapped to its same-name in1 channel via `initializeState()`
- Updated button objectName format to four fields: `outGroup|outputChannel|inputGroup|sourceChannel`
- Updated `syncFromKnob` parsing to handle four-field objectNames

## Decisions Made

- Radio no-op guard placed immediately after null check ‚Äî unchecked events fully suppressed so rows are always filled
- `outGroup` captured from `groupLabel` in `buildGroup` lambda and stored as first field of objectName ‚Äî enables per-group radio scope without a separate metadata table
- `const` buttons use inputGroup `"in1"` by convention ‚Äî `const:0` and `const:1` are treated as virtual in1 sources in serialization (`in1:const:0` / `in1:const:1`)
- `in2StartCol` is always `in1Count+3` regardless of `in2Active` ‚Äî const columns and spacer always present, in2 buttons disabled when in2=none
- `initializeState()` does not call `changed()` ‚Äî avoids recursive rebuild when setting identity routing on first open
