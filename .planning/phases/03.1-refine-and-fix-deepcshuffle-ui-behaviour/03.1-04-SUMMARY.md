# Phase 03.1 Plan 04: Embedded ChannelSet Pickers Summary

**NDK ChannelSet knob rows hidden with Knob::INVISIBLE, replaced by QComboBox pickers embedded inside the shuffle matrix widget header rows above/right of column and row groups**

## Accomplishments
- Added `Knob::INVISIBLE` to all four `Input_ChannelSet_knob` `SetFlags` calls in `DeepCShuffle2::knobs()` — knob rows no longer appear above the matrix in the Nuke panel; values still serialize to .nk
- Added `QComboBox*` members (`_in1Picker`, `_in2Picker`, `_out1Picker`, `_out2Picker`) to `ShuffleMatrixWidget` with correct lifecycle (initialized to nullptr, nulled in clearLayout, created fresh in buildLayout)
- Implemented `populatePicker()`: fills combo with preset layer names and dynamically adds unlisted layers, using `QSignalBlocker` RAII to prevent re-entrant signals during population
- Implemented `onPickerChanged()`: calls `set_text()` then explicit `knob_changed()` on the target NDK knob, guaranteeing matrix column/row rebuild fires when user selects a new layer
- Extended `buildGroup` lambda with `outPickerRef`, `outPickerKnobName`, `outPickerChannelSet` parameters to inject per-group out pickers at `outPickerCol` (one column right of the group label)

## Decisions Made
- `Knob::INVISIBLE` keeps knob rows hidden from the panel but preserves NDK serialization — `KNOB_CHANGED_ALWAYS` ensures `knob_changed()` still fires when the widget writes back via `set_text()`, so the matrix rebuilds correctly.
- QComboBox pickers are recreated on each `buildLayout()` call (not re-parented): `clearLayout()` deletes all child widgets, so pointers are nulled there and freshly created in `buildLayout()`. Simpler than re-parenting.
- `onPickerChanged` calls `knob_changed()` explicitly after `set_text()` as a safety measure — `set_text()` may or may not auto-fire `knob_changed()` on `Input_ChannelSet_knob`, and the explicit call is a no-op if it already fired.
