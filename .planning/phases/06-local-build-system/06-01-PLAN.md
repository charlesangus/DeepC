Create docker-build.sh at the repo root with the following structure and exact implementation:

**Header and defaults:**
```bash
#!/bin/bash
set -euo pipefail
```

**Default version array (LOCKED DECISION -- only 16.0, no 16.1 or 17.0):**
```bash
NUKE_VERSIONS=("16.0")
```

**Default platform flags:**
```bash
BUILD_LINUX=true
BUILD_WINDOWS=true
```

**Docker image registry constant:**
```bash
DOCKER_REGISTRY="ghcr.io/gillesvink/nukedockerbuild"
```

**Nuke SDK path constant (CRITICAL -- FindNuke.cmake uses NO_SYSTEM_ENVIRONMENT_PATH so CMAKE_PREFIX_PATH is ignored; must pass Nuke_ROOT explicitly):**
```bash
NUKE_SDK_PATH="/usr/local/nuke_install"
```

**Argument parsing -- while loop with case statement:**
- `--versions` followed by a comma-separated string: `IFS=',' read -ra NUKE_VERSIONS <<< "$2"` then `shift 2`
- `--linux`: sets `BUILD_LINUX=true`, `BUILD_WINDOWS=false`, then `shift`
- `--windows`: sets `BUILD_LINUX=false`, `BUILD_WINDOWS=true`, then `shift`
- `--all`: sets `BUILD_LINUX=true`, `BUILD_WINDOWS=true`, then `shift`
- `--help` / `-h`: prints usage summary and exits 0
- `*`: prints "Unknown flag: $1" to stderr and exits 1

**Usage function (printed by --help):**
```
Usage: docker-build.sh [OPTIONS]
  --versions VER[,VER...]   Override default Nuke versions (default: 16.0)
  --linux                    Build Linux artifacts only
  --windows                  Build Windows artifacts only
  --all                      Build both platforms (default)
  -h, --help                 Show this help
```

**Prerequisite check:**
- Verify `docker` is available: `if ! command -v docker &>/dev/null; then echo "ERROR: docker is not installed or not in PATH" >&2; exit 1; fi`
- Verify `zip` is available: same pattern

**Directory creation:**
```bash
BASEDIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
mkdir -p "${BASEDIR}/build" "${BASEDIR}/install" "${BASEDIR}/release"
```

**Main loop -- iterate over NUKE_VERSIONS array:**
For each `nuke_version` in `"${NUKE_VERSIONS[@]}"`:

1. Print status: `echo "=== Building DeepC for Nuke ${nuke_version} ==="`

2. **Linux build block (if BUILD_LINUX=true):**
   - Image tag: `${DOCKER_REGISTRY}:${nuke_version}-linux-slim-latest`
   - Build dir (inside container): `/nuke_build_directory/build/${nuke_version}-linux`
   - Install dir (inside container): `/nuke_build_directory/install/${nuke_version}-linux`
   - Docker invocation:
     ```bash
     docker run --rm \
         -v "${BASEDIR}:/nuke_build_directory" \
         "${DOCKER_REGISTRY}:${nuke_version}-linux-slim-latest" \
         bash -c "
             cmake -S /nuke_build_directory \
                   -B /nuke_build_directory/build/${nuke_version}-linux \
                   -D CMAKE_INSTALL_PREFIX=/nuke_build_directory/install/${nuke_version}-linux \
                   -D Nuke_ROOT=${NUKE_SDK_PATH} &&
             cmake --build /nuke_build_directory/build/${nuke_version}-linux --config Release &&
             cmake --install /nuke_build_directory/build/${nuke_version}-linux
         "
     ```
   - Zip creation (after docker run exits successfully):
     ```bash
     zip -r "${BASEDIR}/release/DeepC-Linux-Nuke${nuke_version}.zip" "${BASEDIR}/install/${nuke_version}-linux"
     ```
   - Print: `echo "  Linux build complete: release/DeepC-Linux-Nuke${nuke_version}.zip"`

3. **Windows build block (if BUILD_WINDOWS=true):**
   - Image tag: `${DOCKER_REGISTRY}:${nuke_version}-windows-latest`
   - Build dir: `/nuke_build_directory/build/${nuke_version}-windows`
   - Install dir: `/nuke_build_directory/install/${nuke_version}-windows`
   - Docker invocation (CRITICAL: escape `$GLOBAL_TOOLCHAIN` as `\$GLOBAL_TOOLCHAIN` so it expands inside the container, not on the host):
     ```bash
     docker run --rm \
         -v "${BASEDIR}:/nuke_build_directory" \
         "${DOCKER_REGISTRY}:${nuke_version}-windows-latest" \
         bash -c "
             cmake -S /nuke_build_directory \
                   -B /nuke_build_directory/build/${nuke_version}-windows \
                   -GNinja \
                   -DCMAKE_SYSTEM_NAME=Windows \
                   -DCMAKE_BUILD_TYPE=Release \
                   -DCMAKE_TOOLCHAIN_FILE=\$GLOBAL_TOOLCHAIN \
                   -D CMAKE_INSTALL_PREFIX=/nuke_build_directory/install/${nuke_version}-windows \
                   -D Nuke_ROOT=${NUKE_SDK_PATH} &&
             cmake --build /nuke_build_directory/build/${nuke_version}-windows --config Release &&
             cmake --install /nuke_build_directory/build/${nuke_version}-windows
         "
     ```
   - Zip creation:
     ```bash
     zip -r "${BASEDIR}/release/DeepC-Windows-Nuke${nuke_version}.zip" "${BASEDIR}/install/${nuke_version}-windows"
     ```
   - Print: `echo "  Windows build complete: release/DeepC-Windows-Nuke${nuke_version}.zip"`

**Final summary:**
```bash
echo ""
echo "=== Build complete ==="
echo "Archives in: ${BASEDIR}/release/"
ls -la "${BASEDIR}/release/"*.zip 2>/dev/null || echo "  (no archives produced)"
```

**Make executable:** After creating the file, run `chmod +x docker-build.sh`.

**Important implementation notes:**
- Use separate build dirs per platform (`build/16.0-linux/` vs `build/16.0-windows/`) to avoid CMake cache conflicts (Pitfall 2 from research)
- Use separate install dirs per platform (`install/16.0-linux/` vs `install/16.0-windows/`) to avoid artifact collisions (Pitfall 5 from research)
- ALWAYS pass `-D Nuke_ROOT=/usr/local/nuke_install` -- FindNuke.cmake uses NO_SYSTEM_ENVIRONMENT_PATH (Pitfall 1 from research)
- NEVER expand `$GLOBAL_TOOLCHAIN` on host -- escape it (Pitfall 3 from research)
- Use `zip` not `tar.gz` -- matches batchInstall.sh convention (LOCKED DECISION)
