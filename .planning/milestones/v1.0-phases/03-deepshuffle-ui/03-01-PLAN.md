<task type="auto">
  <name>Task 2: Add Qt6 find_package and AUTOMOC to src/CMakeLists.txt</name>
  <files>src/CMakeLists.txt</files>
  <action>
Read src/CMakeLists.txt in full first. Then make the following targeted additions:

1. After the `include_directories(${NUKE_INCLUDE_DIRS})` line (line ~60), add:

```cmake
# Qt6 — required for ShuffleMatrixKnob embedded Qt widget (Phase 03)
find_package(Qt6 COMPONENTS Core Gui Widgets)
if (Qt6_FOUND)
  set(CMAKE_AUTOMOC ON)
endif()
```

2. After the `# link DeepCPMatte with OpenGL` block (near bottom, after the foreach loops), add:

```cmake
# Link Qt6 into DeepCShuffle for ShuffleMatrixKnob widget
if (Qt6_FOUND)
  target_sources(DeepCShuffle PRIVATE ShuffleMatrixWidget.h)
  target_link_libraries(DeepCShuffle PRIVATE Qt6::Core Qt6::Gui Qt6::Widgets)
endif()
```

Note: Use `Qt6::Core` (with the version suffix), NOT `Qt::Core` — the NDK example uses the unversioned alias which may not be defined on this system. If `Qt6::Core` does not link, try `Qt::Core` as fallback. The key difference from the NDK example CMakeLists: we must add `target_sources(DeepCShuffle PRIVATE ShuffleMatrixWidget.h)` so AUTOMOC sees the Q_OBJECT file; this line is safe to have even before ShuffleMatrixWidget.h exists, but CMake reconfigure after the file is created will be needed anyway.

Do NOT place `set(CMAKE_AUTOMOC ON)` at the top of the file before any targets are defined — it must come after `find_package(Qt6)` succeeds so it only activates when Qt6 is available.

After editing, run:
```bash
cd /workspace && cmake -S . -B build
```
to reconfigure. Expected output: "Found Qt6: ..." line in cmake output. Verify no errors.
  </action>
  <verify>

  </verify>
