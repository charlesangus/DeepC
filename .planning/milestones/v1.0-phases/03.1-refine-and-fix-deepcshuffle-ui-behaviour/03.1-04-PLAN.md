**Important:** The `_in1Picker`, `_in2Picker`, `_out1Picker`, `_out2Picker` pointers are widget-lifetime (created once, reused across rebuilds). Check `if (!_inXPicker)` before creating. On rebuild (clearLayout + buildLayout), the pointers remain valid since `clearLayout()` uses `_gridLayout->takeAt()` which removes from layout without deleting. Re-add them to the new layout positions in each `buildLayout()` call.

**Wait — clearLayout() deletes child widgets:** The current `clearLayout()` calls `delete item->widget()`. This would delete the pickers. Two options:
1. Re-parent pickers to survive clearLayout: `picker->setParent(nullptr)` before clearLayout, then `picker->setParent(this)` after.
2. Recreate pickers on each buildLayout call (simpler, no re-parenting needed).

**Use option 2 (recreate):** Set the four picker pointers to `nullptr` in `clearLayout()` (they are already deleted by the layout cleanup). `buildLayout()` checks `if (!picker)` and creates them fresh each time. This is simple and correct.

**populatePicker implementation:**
```cpp
void ShuffleMatrixWidget::populatePicker(QComboBox* picker,
                                          const std::string& currentLayerName)
{
    const QSignalBlocker blocker(picker);
    picker->clear();
    // Common layer names. In a production implementation, these would be
    // populated dynamically from the Op's available channels.
    const std::vector<std::string> knownLayers = {
        "none", "rgba", "depth", "motion", "forward", "backward",
        "normals", "diffuse", "specular", "emission", "shadow"
    };
    for (const std::string& layerName : knownLayers)
        picker->addItem(QString::fromStdString(layerName));

    // Set current selection.
    const std::string displayName = currentLayerName.empty() ? "none" : currentLayerName;
    const int index = picker->findText(QString::fromStdString(displayName));
    if (index >= 0)
        picker->setCurrentIndex(index);
    else
    {
        // Layer not in preset list — add it and select it.
        picker->addItem(QString::fromStdString(displayName));
        picker->setCurrentIndex(picker->count() - 1);
    }
}
```

**onPickerChanged implementation:**
```cpp
void ShuffleMatrixWidget::onPickerChanged(const std::string& knobName,
                                           const std::string& selectedLayer)
{
    if (!_knob) return;
    DD::Image::Knob* targetKnob = _knob->op()->knob(knobName.c_str());
    if (!targetKnob) return;

    const std::string layerValue = (selectedLayer == "none") ? "" : selectedLayer;
    targetKnob->set_text(layerValue.c_str());
    // Explicitly notify the Op that the knob changed, triggering knob_changed().
    _knob->op()->knob_changed(targetKnob);
}
```

**Note on set_text format:** NDK's `Input_ChannelSet_knob` with `set_text("rgba")` sets the ChannelSet to `Mask_RGBA`. Test at runtime — if the matrix does not update, fall back to calling `set_text` with the full channel name string (e.g. "rgba.red rgba.green rgba.blue rgba.alpha"). If `knob_changed` does not fire automatically via `set_text`, the explicit `_knob->op()->knob_changed(targetKnob)` call ensures the update happens.
