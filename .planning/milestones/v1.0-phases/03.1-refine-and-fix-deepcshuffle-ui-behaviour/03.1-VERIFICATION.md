# Phase 03.1: Refine and Fix DeepCShuffle UI Behaviour — Verification Report

**Phase Goal:** DeepCShuffle2 has full visual and behavioral parity with the reference Shuffle2 node: colored X-mark channel buttons, embedded ChannelSet pickers, radio enforcement scoped to each output group, const:0/1 source columns, identity routing on first open, and disabled-not-hidden behavior for none ChannelSets.

**Note on goal deviation:** The "embedded ChannelSet pickers" goal item was revised during UAT in plan 04. A QComboBox approach was implemented then reverted due to a segfault when calling set_text()+knob_changed() from within a currentTextChanged signal handler inside Nuke's panel refresh cycle. The final implementation uses native NDK ChannelSet knobs ordered above and below the matrix widget (in1/in2 above, out1/out2 below), which provide equivalent functionality without the write-back hazard. This is a legitimate scope reduction documented in the plan 04 SUMMARY.

### Human Verification Required

The automated checks confirm that every behavioral mechanism is correctly implemented in code. The following 10 items require runtime verification in Nuke because they depend on the Nuke panel rendering pipeline, Qt event dispatch, or visual inspection:

#### 1. Radio No-op

**Test:** Place a DeepCShuffle2 node. Open the panel. Click a button that is already checked.
**Expected:** The button stays checked. The routing state does not change.
**Why human:** Requires Qt `QPushButton::toggled(false)` event to be dispatched through Nuke's embedded widget panel.

#### 2. Radio Scope Isolation

**Test:** Set out1 ChannelSet to rgba and out2 ChannelSet to rgba. Route out1.red to the green column. Then route out2.red to the blue column.
**Expected:** out1.red remains routed to green after the out2 change.
**Why human:** Requires two separate output groups with identically-named channels to exist simultaneously.

#### 3. const:0 / const:1 Render Values

**Test:** Route an output channel to const:0. Connect to a Viewer and render.
**Expected:** The routed output channel has value 0.0 (black). Routing to const:1 produces 1.0 (white).
**Why human:** Requires Nuke's Deep render pipeline to execute `doDeepEngine`.

#### 4. Identity Routing on New Node

**Test:** Place a fresh DeepCShuffle2 node with in1=rgba. Open the panel for the first time (state is empty).
**Expected:** Each output row shows its matching input pre-checked (rgba.red → in1:rgba.red, etc.).
**Why human:** Requires a node with no persisted `_matrixState` to trigger the identity-init path.

#### 5. Disabled-not-hidden for None ChannelSet

**Test:** Set in2 ChannelSet to none (Chan_Black). Observe the matrix widget.
**Expected:** in2 columns are visible in the layout but all buttons are greyed out and cannot be clicked. The column group does not collapse.
**Why human:** When in2=Chan_Black, `in2Columns` is empty so `in2Count=0` — no in2 buttons are rendered. This is the code's current behavior. The "disabled-not-hidden" requirement applies only when a ChannelSet transitions from having channels to Chan_Black. Runtime test needed to confirm whether placeholder disabled columns are shown or the group simply has zero buttons.

#### 6. Single-Step ChannelSet Update

**Test:** Change the in1 ChannelSet picker. Observe the matrix widget column headers.
**Expected:** Column headers update immediately on the first change without requiring a second interaction.
**Why human:** Requires observing the `knob_changed` → `setChannelSets` → `updateWidgets` → `syncFromKnob` chain in Nuke's live panel.

#### 7. Colored Buttons and X-mark

**Test:** Open the DeepCShuffle2 panel with rgba ChannelSets. Inspect the toggle buttons.
**Expected:** Red channel buttons show `QColor(180,45,45)`. Green shows `QColor(45,140,45)`. Blue shows `QColor(45,90,180)`. Alpha shows `QColor(155,155,155)`. A checked button displays a white X mark. An unchecked button shows the channel color without an X.
**Why human:** `paintEvent` rendering must be visually confirmed; cannot be verified by code inspection alone.

#### 8. Arrow Decorators

**Test:** Open the DeepCShuffle2 panel. Compare to reference/shuffle.png.
**Expected:** Down-arrows (↓) appear below each column header name in a dedicated row. Right-arrows (→) prefix each output row label.
**Why human:** Visual layout inspection required.

#### 9. NDK ChannelSet Knob Layout

**Test:** Open the DeepCShuffle2 panel.
**Expected:** in1 and in2 knob rows appear above the matrix widget. out1 and out2 knob rows appear below. All four rows are visible (not hidden by INVISIBLE flag).
**Why human:** Requires Nuke panel rendering to verify knob ordering and visibility.

#### 10. Script Save/Load Round-Trip

**Test:** Make routing changes in the matrix, save the .nk script, close Nuke, reopen the script.
**Expected:** All four ChannelSet selections and the full matrix routing state are restored exactly as saved. The `matrix` knob value and in1/in2/out1/out2 values are present in the .nk file.
**Why human:** Requires Nuke script I/O cycle to exercise `to_script`/`from_script` and NDK knob serialization.
