# Phase 03.1 Plan 04: UAT Fix Summary

**Native NDK ChannelSet knobs restored as visible and correctly ordered around the matrix widget; inter-group column spacer gap removed after QComboBox pickers segfaulted during UAT**

## Performance

## Accomplishments

- Removed all QComboBox picker code from ShuffleMatrixWidget.h and .cpp — the approach caused runtime segfaults when selecting a layer and showed incorrect channel info
- Restored NDK ChannelSet knobs to visible by removing the INVISIBLE flag; kept KNOB_CHANGED_ALWAYS so knob_changed() still fires on any change
- Fixed knob panel ordering: in1/in2 registered before CustomKnob2 (appear above the matrix); out1/out2 registered after CustomKnob2 (appear below the matrix)
- Removed the explicit 12px spacer column between the const columns and in2 column group — QGridLayout's 2px uniform spacing is sufficient and the dedicated column was causing a visible gap

## Task Commits

1. **Task 1: Hide NDK knobs with INVISIBLE and embed QComboBox pickers in widget** - `8785a83` (feat) — original attempt, failed UAT
2. **Task 2 (UAT Fix): Revert QComboBox, restore visible knobs, fix knob ordering, remove column gap** - `d0605c9` (fix)

## Files Created/Modified

- `src/DeepCShuffle2.cpp` — Removed INVISIBLE flag from all four SetFlags calls; reordered knobs() so in1/in2 precede CustomKnob2 and out1/out2 follow it
- `src/ShuffleMatrixWidget.h` — Removed QComboBox include, four QComboBox* members, populatePicker and onPickerChanged declarations
- `src/ShuffleMatrixWidget.cpp` — Removed all QComboBox construction/population/signal code; removed spacerCol from column layout computation; restored plain QLabel "in 1"/"in 2" group headers

## Decisions Made

- QComboBox pickers reverted entirely: calling set_text() + knob_changed() from inside a currentTextChanged signal handler fired during Nuke's UI refresh cycle causes a segfault. The embedded widget write-back pattern is unsafe in this context. Native NDK ChannelSet knob rows are robust and integrate cleanly with Nuke's undo/redo and script serialization.
- NDK panel ordering: knobs() registration order is strictly top-to-bottom in the Nuke panel. Placing in1/in2 before the matrix widget and out1/out2 after achieves the closest achievable layout to the reference image. Sideways placement of out knobs beside the matrix is not possible via the standard NDK knob API.
- Spacer column removal: setColumnMinimumWidth(spacerCol, 12) added a dedicated empty column between the const and in2 column groups. This was visible as a gap in the rendered widget. The 2px QGridLayout spacing set on the layout is sufficient for visual separation between all column groups.

## Second UAT Round — Additional Fixes (2026-03-16)

Six additional failures reported after the first UAT pass were fixed in a second round.
All fixes were committed atomically. Full build verified clean after all changes.

### Second-Round UAT Fixes

**FAIL 1 — Disabled-not-hidden (commit 9cc41e0)**
- **Issue:** When in2=Chan_Black, forEach over Chan_Black yields zero channels; in2Columns was
  empty so the entire in2 section disappeared from the widget.
- **Fix:** After iterating in2Set, if in2Disabled and in2Columns is still empty, insert placeholder
  names (rgba.red/green/blue/alpha) so the column group always renders. All placeholder buttons
  receive setEnabled(false) via the existing in2Disabled path.
- **Files:** src/ShuffleMatrixWidget.cpp

**FAIL 2 — Radio scope: out1 and out2 linked (commit 9602ddb)**
- **Issue:** buildGroup lambda used groupLabel (human-readable layer name like "rgba") as the
  capturedOutGroup for button objectNames. If both out1 and out2 resolve to the same layer name,
  they shared radio scope and toggling in out1 also untoggled out2 buttons.
- **Fix:** Added separate groupId parameter ("out1"/"out2") to buildGroup lambda; button objectNames
  and onCellToggled calls use groupId for radio scoping. groupLabel is display-only.
- **Files:** src/ShuffleMatrixWidget.cpp

**FAIL 3 — Single-step ChannelSet update lag (commit a52f71a)**
- **Issue:** knob_changed() called updateWidgets() which posts kUpdateWidgets asynchronously.
  The widget rebuild ran with the ChannelSets from the PREVIOUS change, not the current one.
- **Fix:** Added ShuffleMatrixKnob::syncWidgetNow() that calls syncFromKnob() directly on the
  live widget via a stored _widget pointer. knob_changed() now calls syncWidgetNow() synchronously
  after setChannelSets() instead of relying on the async updateWidgets() queue. Added
  clearWidgetPointer() called from kDestroying to prevent use-after-free.
- **Files:** src/DeepCShuffle2.cpp, src/ShuffleMatrixKnob.h, src/ShuffleMatrixKnob.cpp,
  src/ShuffleMatrixWidget.cpp

**FAIL 4 — Layout restructure: embedded QComboBox pickers (commit d024e1d)**
- **Issue:** Target layout (in1/in2 pickers above their columns, out1/out2 pickers right-aligned
  next to their row groups) not achievable with NDK knob ordering.
- **Fix:** Re-embedded QComboBox pickers inside ShuffleMatrixWidget using QTimer::singleShot(0)
  in currentTextChanged handlers to break the re-entrant call stack that caused the original
  segfault. NDK ChannelSet knobs marked INVISIBLE. availableLayerNames() method added to
  ShuffleMatrixKnob to populate pickers without calling back into the Op. Layout restructured:
  row 0 = in1/in2 pickers; groups start at row 1; outLabelCol holds output picker spanning
  group header + data rows.
- **Files:** src/DeepCShuffle2.cpp, src/ShuffleMatrixKnob.h, src/ShuffleMatrixKnob.cpp,
  src/ShuffleMatrixWidget.h, src/ShuffleMatrixWidget.cpp

**FAIL 5 — G/B column gap still present (commit 06d52ae)**
- **Issue:** Remaining column gap between button column groups. QGridLayout defaulting to stretch
  behavior on some columns.
- **Fix:** Replaced setSpacing(2) with explicit setHorizontalSpacing(2) + setVerticalSpacing(2) +
  setContentsMargins(4,4,4,4). Added explicit setColumnStretch(col, 0) for all button columns and
  setColumnStretch(outLabelCol, 1) for the label column.
- **Files:** src/ShuffleMatrixWidget.cpp

**FAIL 6 — White X-mark invisible on const:1 (commit bfa34a9)**
- **Issue:** const:1 button has near-white fill (220,220,220); the hardcoded white X-mark
  pen was invisible against it.
- **Fix:** In ChannelButton::paintEvent, after computing fillColor, check lightnessF() > 0.5
  and use Qt::black for the X pen on light backgrounds, Qt::white otherwise.
- **Files:** src/ShuffleMatrixWidget.cpp

**Arrow size (commit 9af1c62)**
- **Issue:** Small ↓ and → arrows were hard to read at compact column widths.
- **Fix:** makeArrowLabel() lambda sets font pointSize + 4; switched to ⇓ (U+21D3, double
  stroke) for down arrows and ⇒ (U+21D2) for right arrows. Output channel labels get +2 pt.
- **Files:** src/ShuffleMatrixWidget.cpp
