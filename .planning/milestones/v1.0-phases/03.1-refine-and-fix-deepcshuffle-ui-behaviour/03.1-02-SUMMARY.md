# Phase 03.1 Plan 02: Disabled-not-hidden and make_widget Timing Fix Summary

**None ChannelSets now render greyed-out buttons instead of collapsing; first panel open shows correct column headers without a second ChannelSet change**

## Accomplishments
- `buildGroup` lambda extended with `buttonsDisabled` parameter — when out2=none, all out2 row buttons and labels are created but immediately disabled via `setEnabled(false)`
- In2 column buttons use a separate `in2Disabled` flag captured from the outer `buildLayout` scope — correctly disables in2 source buttons without coupling to the row-group disabled state
- `make_widget()` now calls `widget->syncFromKnob()` after constructing the widget — first panel open populates column headers from live ChannelSets instead of Chan_Black defaults

## Task Commits

1. **Task 1: Disabled-not-hidden for none ChannelSets in buildLayout** - `cfaef45` (feat)
2. **Task 2: Fix make_widget timing so first ChannelSet change updates matrix immediately** - `67f03ec` (fix)

## Files Created/Modified
- `src/ShuffleMatrixWidget.cpp` - buildLayout: replaced in2Active/out2Active conditional guards with in2Disabled/out2Disabled flags; buildGroup accepts buttonsDisabled; all buttons and output labels receive setEnabled(!buttonsDisabled)
- `src/ShuffleMatrixKnob.cpp` - make_widget: constructs widget then calls syncFromKnob() before returning

## Decisions Made
- `buttonsDisabled` parameter on `buildGroup` controls in1/const/outLabel buttons; in2 buttons use the outer-scope `in2Disabled` flag directly — this means in2 buttons can be disabled independently of whether the row group itself is disabled (correct behavior: in2 can be none while out2 is active)
- The out2 group guard was changed from `if (out2Active && !out2Rows.empty())` to `if (!out2Rows.empty())` — when out2=Chan_Black it has no channels so `out2Rows` is empty anyway; the `out2Disabled` flag still controls button state correctly
- `make_widget` now triggers two `buildLayout` calls (one in constructor, one via `syncFromKnob`) — this is intentional and harmless; the second call ensures ChannelSets are current on first open
