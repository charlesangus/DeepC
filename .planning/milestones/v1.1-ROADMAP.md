## Milestone Summary

**Key Decisions:**

- `NUKE_VERSIONS` defaults to `("16.0")` only — research confirmed 16.0 is the stable target; 16.1 and 17.0 deliberately excluded from defaults (but supported via `--versions`)
- zip archives over tar.gz — matches existing `batchInstall.sh` convention
- Separate `build/${version}-{platform}` and `install/${version}-{platform}` dirs per invocation — prevents CMake cache conflicts when building both Linux and Windows in a single run
- `Nuke_ROOT` passed explicitly via `-D` flag — `FindNuke.cmake` uses `NO_SYSTEM_ENVIRONMENT_PATH` so `CMAKE_PREFIX_PATH` is ignored
- `$GLOBAL_TOOLCHAIN` escaped as `\\$GLOBAL_TOOLCHAIN` in `bash -c` string — evaluated by the container shell, not the host

**Issues Resolved:**

- NukeDockerBuild image tag format corrected (dropped `-slim-latest` suffix)
- Windows Qt6 support added (Qt6 path inside Windows container differs from Linux)
- Auto-bootstrap of NukeDockerBuild images from source when prebuilt images are unavailable
- Zip archive structure fixed to place `.so`/`.dll` at archive root

**Issues Deferred:**

- GitHub Actions CI/CD — shelved; local builds cover the immediate need

- None identified
