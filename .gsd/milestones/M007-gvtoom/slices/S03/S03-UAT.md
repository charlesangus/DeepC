2. Nuke 16.0 is available with DeepCDefocusPORay loaded (plugin path includes the .so).
3. Angenieux 55mm lens files are accessible:

   - `aperture.fit` (path known to tester)
4. `test/test_ray.nk` is accessible (checked into source tree).
5. A 128×72 checkerboard Deep image can be generated by the test script's Reformat→Grid→Dilate→Merge→DeepFromImage DAG (self-contained, no external input required).

## TC-01: End-to-end test script exits 0 with non-black output

**Goal:** Confirm `nuke -x test/test_ray.nk` completes successfully and writes a non-black EXR.

**Steps:**

1. Open a terminal in the project root.
2. Run: `nuke -x test/test_ray.nk`
3. After completion, check that `test/test_ray.exr` exists.
4. Run: `python3 -c "import OpenEXR, Imath; f=OpenEXR.InputFile('test/test_ray.exr'); r=f.channel('R',Imath.PixelType(Imath.PixelType.FLOAT)); import struct; vals=struct.unpack(f'{len(r)//4}f',r); print(sum(1 for v in vals if v>0))"`
5. Note the count of non-zero red pixels.

**Expected:** `nuke -x` exits 0; `test_ray.exr` exists; non-zero red pixel count is **≥ 100**.

**Failure signal:** Count < 100 or exit code ≠ 0 → poly files not loading, aperture_file path invalid, or gather selectivity guard rejecting all samples. Check Nuke console for `error()` messages about lens file load failure.

## TC-02: Defocused output is visually distinct from the input (not a ring or flat color)

**Goal:** Confirm the gather engine produces recognisable defocus bokeh, not an aperture ring or a black image.

**Steps:**

1. Open Nuke and load `test/test_ray.nk`.
2. View the DeepCDefocusPORay1 node output (Connect to Viewer).
3. Compare against the DeepFromImage node output (view source Deep-to-flat as a reference checkerboard).

**Expected:** The output shows a soft, defocused version of the checkerboard pattern — blurred/smeared squares, not sharp edges. Output is not all-black, not a bright ring or circle, and not identical to the undefocused input.

**Failure signal:** All black → poly not loaded. Ring/circle pattern → gather selectivity guard not firing. Sharp identical image → aperture_samples = 0 or no deep samples in bounds.

## TC-03: Holdout input attenuates output correctly (sharp mask, not blurred)

**Goal:** Confirm the holdout transmittance mechanism works and produces a sharp, undefocused holdout boundary.

**Steps:**

1. In Nuke, duplicate the DeepFromImage node and connect it to the DeepCDefocusPORay1 node's second (holdout) input.
2. Mask the holdout to approximately the left half of the frame (e.g. using a DeepCGrade to zero the right half).
3. View the DeepCDefocusPORay1 output.

**Expected:** The left half of the output is substantially darker or fully attenuated (the holdout blocks contribution). The attenuated boundary is **sharp**, not blurred. The right half retains defocused output. No defocus is applied to the holdout geometry itself.

**Failure signal:** Blurred holdout boundary → holdout evaluated at input pixel position instead of output pixel. No attenuation → `transmittance_at` lambda not applied or `holdoutConnected` always false.

## TC-04: Chromatic aberration produces per-channel color fringing

**Goal:** Confirm that wavelength-specific poly evaluation (0.45/0.55/0.65 μm) produces visible chromatic aberration in the bokeh.

**Steps:**

1. Load `test/test_ray.nk` in Nuke and view the output.
2. Zoom into a high-contrast bokeh region (e.g. bright checkerboard square against dark background).
3. Toggle between R, G, and B channel views on the Viewer.

**Expected:** The bokeh circle/soft disc appears at slightly different positions or radii in R vs. G vs. B channels. The color fringe is subtle (a few pixels) at the default Angenieux 55mm settings but should be measurable by overlaying the channels.

**Failure signal:** R/G/B bokeh at identical positions → all three channels using the same wavelength, or CA loop collapsed to single evaluation.

## TC-05: max_degree knob changes render output

**Goal:** Confirm `_max_degree` is passed to both poly evaluations and that changing it affects the bokeh character.

**Steps:**

1. Load `test/test_ray.nk` in Nuke, render at default `max_degree=11`, save output.
2. Change `max_degree` to 3 on the DeepCDefocusPORay1 node.
3. Re-render, compare to the max_degree=11 output.

**Expected:** The two renders are visually distinguishable. Degree 11 shows more complex bokeh aberrations (higher-order terms active); degree 3 shows a cleaner, more symmetric bokeh. Neither should be all-black.

**Failure signal:** Both renders are identical → `_max_degree` not passed to `poly_system_evaluate`, or `poly_system_evaluate` ignoring the degree parameter. Degree 3 render is all-black → early exit is eliminating all terms (degree threshold logic inverted).

## TC-06: Aperture vignetting rejects samples outside aperture housing radius

**Goal:** Confirm the aperture.fit vignetting check correctly gates contributions.

**Steps:**

1. Load the node in Nuke with a valid `aperture_file` knob pointing to `aperture.fit`.
2. Reduce `fstop` to a very low value (e.g. 1.4) to maximise aperture size.
3. Note the output brightness/coverage.
4. Increase `fstop` to 22 (very small aperture).
5. Compare output to fstop 1.4.

**Expected:** High fstop (f/22) produces a dimmer output than low fstop (f/1.4) because fewer aperture samples pass the housing radius check. The output should not be completely black at any fstop — some samples should always pass through the aperture.

**Failure signal:** All-black at any fstop → aperture housing radius check too strict or aperture_housing_radius knob value incorrect for this lens. Identical output regardless of fstop → vignetting check not applied.

---

## TC-07: Missing aperture_file produces error() message (not silent crash)

**Goal:** Confirm graceful degradation when `aperture_file` points to a missing file.

**Steps:**

1. In Nuke, set the `aperture_file` knob on DeepCDefocusPORay1 to a path that does not exist (e.g. `/tmp/nonexistent.fit`).
2. Trigger a render.
3. Observe the Nuke Script Editor / error panel.

**Expected:** Nuke displays an error message containing the failing file path (e.g. `aperture poly open failed: /tmp/nonexistent.fit`). The render does not crash Nuke. Output is all-black (zero_output lambda fires because aperture poly not loaded).

**Failure signal:** Nuke crashes (SIGSEGV, malloc error) → poly_system_read called without null-check or at wrong time. No error message displayed → `error()` not wired on aperture load failure path.

---

## TC-08: aperture_samples=1 vs aperture_samples=16 shows expected noise vs quality tradeoff

**Goal:** Confirm Halton+Shirley aperture sampling quality scales with sample count.

**Steps:**

1. Set `aperture_samples` to 1 and render `test/test_ray.nk`.
2. Set `aperture_samples` to 16 and render.
3. Compare the two outputs visually.

**Expected:** The aperture_samples=1 render shows a coarser, noisier bokeh. aperture_samples=16 shows a smoother bokeh disc/shape. Neither is all-black.

**Failure signal:** Both renders identical → Halton sequence not advancing (k not incrementing) or aperture_samples knob not read. Low-sample render has structured patterns instead of stochastic noise → Halton sampling replaced with grid sampling.

## Edge Cases

### EC-01: Empty deep input (no samples in region)
- Set an extremely small `focus_distance` that pushes all samples outside the CoC neighbourhood.
- **Expected:** All-black output (no contributing samples), no crash.

### EC-02: Single-pixel deep input
- Connect a Deep image with a single sample at the center.
- **Expected:** A single defocused disc (or point, depending on CoC size) appears at the expected output position. No crash from empty-neighbourhood loop.

### EC-03: Input deep region smaller than expanded bounds
- Use a small (`32×18`) input deep image rendered into a `128×72` output.
- **Expected:** Output is correctly clipped to the input's coverage. No out-of-bounds read or crash. The expanded Box clamping to `in_box` prevents reads outside the valid input region.

### EC-04: poly_file missing (exitpupil.fit not set)
- Clear the `poly_file` knob.
- **Expected:** `error()` message displayed; all-black output (same as TC-07 but for exitpupil path). No crash.

## Failure Signals Reference

| Symptom | Likely Cause |
|---------|-------------|
| All-black output | poly_file or aperture_file not set/missing; zero input samples |
| Nuke crash (malloc/segfault) | `writableAt` channel index vs enum bug; poly loaded from `_validate` (data race) |
| Aperture ring (not bokeh) | Gather selectivity guard `ox_land != ox` not firing |
| Sharp/undefocused output | `max_coc_px` = 0; input neighbourhood not expanded |
| Identical R/G/B channels | CA wavelength loop collapsed |
| Blurred holdout boundary | Holdout evaluated at input pixel not output pixel |
| f/22 = all-black | `aperture_housing_radius` too small or vignetting inverted |
