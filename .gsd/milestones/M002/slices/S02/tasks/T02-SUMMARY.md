# T02: Verify docker-build.sh and CMake integration for DeepCBlur

**Verified DeepCBlur compiles, installs to release archive with icon and Filter menu entry via full Docker Linux build.**

## What Happened

Ran all five verification steps from the task plan. Steps 1 (PLUGINS list), 2 (FILTER_NODES), and 4 (menu.py.in @FILTER_NODES@) passed via grep. Step 3 (icon glob) revealed that the plan's assumption about `file(GLOB ICONS ...)` was wrong — no such pattern exists in `src/CMakeLists.txt`. Icons are installed via a separate `install(DIRECTORY)` rule in the top-level `CMakeLists.txt`, which picks up all files from the `icons/` directory. Step 5 succeeded: Docker was available with pre-built `nukedockerbuild:16.0-linux` image, so a full Linux build was executed via `docker-build.sh --linux --versions 16.0`. The build completed in ~40 seconds, producing `release/DeepC-Linux-Nuke16.0.zip` containing both `DeepC/DeepCBlur.so` and `DeepC/icons/DeepCBlur.png`. The generated `menu.py` correctly places DeepCBlur in the Filter submenu alongside DeepThinner.

## Verification

All slice-level checks pass (5/5). Additionally, the Docker build provided end-to-end proof that DeepCBlur compiles against the Nuke 16.0 SDK, gets installed, and appears in the release archive with correct menu integration.

This is not a problem; the icon is correctly picked up and included in the release archive.
